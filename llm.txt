====================
八、图片 URL 解析规则
====================

对于 `canvas.canvasItems` 里的图片项，`imagePath` 不能直接假设可用于 `<img src>`。
必须使用下面这个规则转换：

```js
const getImageUrl = (imagePath, canvasName, apiBaseUrl) => {
  let normalized = String(imagePath || "").replace(/\\/g, "/");
  if (normalized.startsWith("/")) {
    normalized = normalized.slice(1);
  }
  if (normalized.startsWith("assets/")) {
    const filename = normalized.split("/").pop() || normalized;
    const safeCanvasName = encodeURIComponent(canvasName || "Default");
    const safeFilename = encodeURIComponent(filename);
    return `${apiBaseUrl}/api/assets/${safeCanvasName}/${safeFilename}`;
  }
  if (normalized.startsWith("http://") || normalized.startsWith("https://")) {
    return normalized;
  }
  return `${apiBaseUrl}/${normalized}`;
};
```

这是图像处理类命令的标准入口。
凡是要加载画布图片、复制图片、裁剪、像素化、拼接、分析、导出，都优先复用这个函数。

====================
九、图片加载标准模式
====================

```js
const loadImage = (src) =>
  new Promise((resolve, reject) => {
    const img = new Image();
    img.crossOrigin = "anonymous";
    img.onload = () => resolve(img);
    img.onerror = () => reject(new Error("Image load failed"));
    img.src = src;
  });
```

凡是需要在 Canvas 上重绘图像，优先使用这套模式。

====================
十、图像处理后的三种标准落地方式
====================

图像处理类命令通常只允许用下面三种落地方式，不要另起炉灶。

1. 下载到本地

适用于“导出 PNG/JPG/WebP”、“下载结果图”。

标准模式：

```js
const downloadBlob = (blob, filename) => {
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
};
```

2. 覆盖当前图片

适用于“处理后替换原图”。

流程：
- 先把处理结果转成 `Blob`
- 上传到 `/api/upload-temp`
- 再调用 `updateCanvasImage`

3. 新增副本到画布

适用于“保留原图，再生成一张结果图”。

流程：
- 先把处理结果转成 `Blob`
- 上传到 `/api/upload-temp`
- 再调用 `addToCanvas`

====================
十一、/api/upload-temp 契约
====================

图像处理后的结果如果要写回画布，标准上传接口如下：

请求：

```js
const uploadTempImage = async (blob, filename, canvasName, apiBaseUrl) => {
  const params = new URLSearchParams();
  if (filename) params.set("filename", filename);
  if (canvasName) params.set("canvasName", canvasName);
  const endpoint = params.size > 0
    ? `${apiBaseUrl}/api/upload-temp?${params.toString()}`
    : `${apiBaseUrl}/api/upload-temp`;
  const headers = blob.type ? { "Content-Type": blob.type } : undefined;
  const res = await fetch(endpoint, {
    method: "POST",
    headers,
    body: blob,
  });
  if (!res.ok) {
    throw new Error(`Upload failed: ${res.status}`);
  }
  return res.json();
};
```

响应结构：

```js
{
  success?: boolean,
  filename?: string,
  path?: string,        // 形如 "assets/xxx.png"
  diskPath?: string,
  width?: number,
  height?: number,
  dominantColor?: string | null,
  tone?: string | null,
}
```

关键点：
- `path` 可直接写回 `imagePath`
- `width` / `height` 可直接写回画布图片尺寸
- `dominantColor` / `tone` 可直接写回图片元数据

====================
十四、前端 Canvas 图像处理标准模式
====================

当命令需要做：
- 裁剪
- 像素化
- 缩放
- 拼接
- 滤镜
- 导出结果图

优先使用这个流程：

1. 取目标图片
2. `getImageUrl(...)`
3. `loadImage(...)`
4. 新建离屏 `<canvas>`
5. 用 2D context 重绘
6. `canvas.toBlob(...)`
7. 根据需求：
   - 下载
   - 覆盖原图
   - 新增副本

`toBlob` 标准包装：

```js
const canvasToBlob = (canvasEl, type = "image/png", quality) =>
  new Promise((resolve, reject) => {
    canvasEl.toBlob((blob) => {
      if (!blob) {
        reject(new Error("Canvas toBlob failed"));
        return;
      }
      resolve(blob);
    }, type, quality);
  });
```

====================
十五、像素级图像处理建议
====================

这是通用图像处理建议，不只针对像素化：

- 不要直接修改原始 `ImageData` 之外的状态
- 对于像素风 / 马赛克 / 缩略图类命令，优先用“先缩小再关闭平滑放大”的方式
- 放大像素风结果时要设置：

```js
ctx.imageSmoothingEnabled = false;
```

- 生成目标图时保留 alpha
- 输出尺寸必须可控，避免无上限大图导致卡死

十六、命令 UI 规则
====================

外部命令是运行时动态加载，Tailwind 不会扫描这里。
如果需要样式，必须在命令内部注入 `<style>`。

可以安全使用这些 CSS 变量：
- `var(--color-primary)`
- `var(--color-neutral-100)`
- `var(--color-neutral-300)`
- `var(--color-neutral-500)`
- `var(--color-neutral-700)`
- `var(--color-neutral-900)`
- `var(--color-white)`

样式注入模板：

```js
const STYLE_ID = "your-command-style";

const STYLE_TEXT = `
.your-command-root {
  display: flex;
  flex-direction: column;
  gap: 12px;
  padding: 16px;
}
`;

const ensureStyle = () => {
  if (document.getElementById(STYLE_ID)) return;
  const style = document.createElement("style");
  style.id = STYLE_ID;
  style.textContent = STYLE_TEXT;
  document.head.appendChild(style);
};
```

在 `ui` 中：

```js
useEffect(() => {
  ensureStyle();
}, []);
```

十七、自动副作用命令的防重规则
====================

如果 `ui` 组件一挂载就自动执行副作用，比如：
- 打开文件夹
- 复制图片

标准做法：

```js
const startedRef = useRef(false);

useEffect(() => {
  if (startedRef.current) return;
  startedRef.current = true;
  void runTask();
}, []);
```

如果还需要更强防重，可以加时间戳节流。

====================
十八、config 必填结构
====================

所有用户可见文案都必须写进命令自己的 `config.i18n`。
至少提供 `en` 和 `zh`。

固定结构：

```js
export const config = {
  id: "yourCommandId",
  i18n: {
    en: {
      "command.yourCommand.title": "Your Command",
      "command.yourCommand.description": "What this command does",
      "toast.command.yourCommand.success": "Done",
      "toast.command.yourCommand.failed": "Failed: {{error}}",
    },
    zh: {
      "command.yourCommand.title": "你的命令",
      "command.yourCommand.description": "命令作用说明",
      "toast.command.yourCommand.success": "已完成",
      "toast.command.yourCommand.failed": "失败：{{error}}",
    },
  },
  titleKey: "command.yourCommand.title",
  title: "Your Command",
  descriptionKey: "command.yourCommand.description",
  description: "What this command does",
  keywords: ["keyword", "命令关键词"],
};
```

要求：
- `id` 用 camelCase
- `titleKey` / `descriptionKey` 必须对应 i18n key
- `title` / `description` 保留英文默认值
- 所有按钮文案、表单文案、toast 文案都必须在 i18n 中

十九、生成策略
====================

1. 纯动作命令
- 优先 `run`
- 例如：新增文本、移出当前编组、固定 URL 打开

3. 图像处理类命令

4. 图像导出类命令
- 明确输出走哪一种标准落地方式：
  - 下载
  - 覆盖原图
  - 新增副本
